### 1.95

#### document Closure capture precision: https://github.com/rust-lang/fls/pull/698

Tshepang: This is about closure capture and I added that the capture only happens when there is a read. Hristian made a suggestion that I think is redundant: https://github.com/rust-lang/fls/pull/698#discussion_r3474062185

Hristian: The original suggestion I made had no mention of reads, ??

Tshepang: The Reference does mention the reads. I don't know why, it feels like an over-explanation to me. Something that should go to the compiler documentation rather than language documentation.

Hristian: It's not necessarily 1:1 when it comes to a particular construct. There's a rule saying that a place is captured by an immutable borrow ??. We don't capture a discriminant but the whole place. FLS ID: fls_Vt9C9mKxHOwo

Hristian: If you can establish there's 1:1 correspondence between a thing that's been captured and the thing that's been read, you can drop the "is read part".

Tshepang: Could I drop the rule 

**Conclusion**: Mentioning that capturing occurs when there is a read, and listing precisely when a read occurs does not add any value to the rules, so we decided to drop the "read" aspect of the rules.
