## FLS 1.91.0, 1.91.1 update

### [[Change]: [1.91.0, 1.91.1] Stabilize 'sse4a' and 'tbm' target features](https://github.com/rust-lang/fls/issues/625)

Tshepang: To avoid updating the changelog multiple times while a PR is evolving, we should write the changelog changes when the PR converges.

### [[Change]: [1.91.0, 1.91.1] Add 'target_env = "macabi"' and 'target_env = "sim"' cfgs as replacements for the `target_abi` cfgs with the same values](https://github.com/rust-lang/fls/issues/626)

Eric: Since the set of values for `target_env` is not closed, we chose not to update the FLS with respect to `macabi` and `sim`.

### [[Change]: [1.91.0, 1.91.1] Lower pattern bindings in the order they're written and base drop order on primary bindings' order](https://github.com/rust-lang/fls/issues/623)

Pete: The "function and closure arguments" portion of "Or-patterns" in the Reference needs to be change to "function and closure parameters". Hristian should do it.

Pete: What is the meaning of "let-bindings" vs "let statement" in the context of "Or-patterns"?

Eric (or TC?): "let-bindings" encompases "let statements", "if let", "while let", and let chains.

Hristian, Pete: We do not have a "let-binding" term defined in the FLS. Regardless, the Or-pattern section should talk about "let-binding"s.

### [[Change]: [1.91.0, 1.91.1] Stabilize declaration of C-style variadic functions for 'sysv64', 'win64', 'efiapi', and 'aapcs' ABIs](https://github.com/rust-lang/fls/issues/624)

Pete, Hristian: Better term for `variadic ABI`?

TC: Seems OK. A `variadic ABI` is an ABI which supports variadic `extern C` functions.

Tshepang: We should take TC's definition, put it in the glossary, but we also need to name the list of applicable ABIs somehow.
